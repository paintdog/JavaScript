Auf dieser Seite werden Variablen behandelt.

Bis zur Zuweisung eines Werts hat die Variable den Wert undefined.

Schlüsselwort Bedeutung
const Es handelt sich um eine Konstante, wobei hier der Wert direkt bei der Deklaration zugewiesen werden muss. Nach der Deklaration kann kein neuer Wert zugewiesen werden. Wie bei let ist auch bei const eine Variable auf ihren Block-Scope beschränkt.
let Gelten nur innerhalb ihres jeweiligen Blocks ({}). Mit let deklarierte Variablen haben einen kleinen Gültigkeitsbereich (scope), was Fehlerquellen reduziert. Wird auf eine mit let angelegte Variable außerhalb ihres Blocks zugegriffen, führt dies zu einem ReferenceError.
var Wenn sie in einer Funktion deklariert wurden, gelten sie dort, wenn sie außerhalb von Funktionen angelegt wurden, sind es globale Variable.
Globale Variablen Variablen, denen ohne ein let, const oder var ein Wert zugewiesen wird, sind implizit/einfach angelegt und damit globale Variablen.

Mit dem Befehl "use strict"; kann ich den Strict Modus aktivieren, der eine Fehlermeldung wirft, wenn Variablen implizit definiert werden.

Variablennamen (camelCase):

  • dürfen Buchstaben, Zahlen und Unterstriche enthalten
  • dürfen nicht mit einer Zahl, einem Sonder- oder Satzzeichen beginnen
  • dürfen keine Leerzeichen oder Bindestriche enthalten
  • Schlüsselwörter sind gesperrt

JavaScript ist case sensitive , d. h. Name, name, NAme sind drei Variablen. Hier würde use strict helfen, entsprechende Falschschreibungen aufzufinden.

Variablennamen, die einen Fehler enthalten, führen zu einem SyntaxError.

JavaScript kennt Gleitkommazahlen, Ganzzahlen (Integer), Strings, Boolean und den Wert null. JavaScript typisiert dynamisch, allerdings gilt die Zuweisung eines anderen Datentyps zu einer Variable als schlechte Programmierpraxis.
